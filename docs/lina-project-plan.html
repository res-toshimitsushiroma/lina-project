1. プロジェクト概要

プロジェクト名

LINA（リナ） - LINE Integrated Appointment System

プロジェクトビジョン

「美容サロンの予約管理を革新し、オーナーとお客様の両方に最高の体験を提供する」

LINEとGoogleの完全統合により、既存のツールを活用しながら、導入コストゼロで始められる予約管理システムを実現します。

背景と課題

  • 市場の課題：既存の予約システムは高額で、小規模サロンには導入障壁が高い
  • 技術的課題：LINE公式アカウントの機能だけでは予約管理が不十分
  • 法的課題：利用規約やトラブル対応の仕組みが不明確
  • 運用課題：顧客情報と予約情報が分散し、一元管理できない

ソリューション

LINE × Google の完全統合プラットフォーム

  • LINE公式アカウント：顧客コミュニケーション
  • Googleカレンダー：スケジュール管理
  • Googleスプレッドシート：データ管理
  • 独自の利用規約管理：法的保護

4. 機能要件

コア機能

1. LINE連携機能

  • LINE公式アカウントAPI連携
  • リッチメニューからの予約導線
  • 自動応答メッセージ
  • 予約確認・リマインド通知
  • キャンセル・変更受付

2. Google連携機能

  • Googleカレンダー同期
  • Googleスプレッドシートへのデータ出力
  • Gmail通知連携
  • Google Drive バックアップ

3. 利用規約管理システム

革新的な全文保存システム

  • 規約の全文スナップショット保存
  • SHA-256ハッシュによる改ざん防止
  • 同意時の技術的証跡記録（IP、デバイス情報）
  • PDF形式での証跡出力
  • 7年間の長期保存

4. 管理画面機能

  • ダッシュボード（予約状況、売上サマリー）
  • 顧客管理（検索、詳細、履歴）
  • メニュー管理
  • スタッフ管理
  • 設定管理

5. システムアーキテクチャ

技術スタック

レイヤー 技術 選定理由
フロントエンド Next.js 14, React, TypeScript, Tailwind CSS 高速、型安全、最新のUI/UX
バックエンド Laravel 10, PHP 8.2 堅牢、豊富なエコシステム
データベース PostgreSQL 15 JSONB対応、高信頼性
認証 Firebase Authentication 安全、スケーラブル
インフラ Google Cloud Run サーバーレス、自動スケール
CDN Cloudflare 高速配信、DDoS対策
監視 Google Cloud Monitoring 統合監視、アラート

セキュリティ設計

  • 認証：Firebase Authentication（多要素認証対応）
  • 通信：全通信SSL/TLS暗号化
  • データ：保存時暗号化（AES-256）
  • API：レート制限、HMAC署名検証
  • 監査：全操作のログ記録

データベース設計（主要テーブル）

-- サロンマスタ
salons (id, name, address, phone, settings)

-- 顧客マスタ  
customers (id, salon_id, name, phone, email, line_user_id)

-- 予約
reservations (id, salon_id, customer_id, menu_id, datetime, status)

-- 利用規約スナップショット
terms_snapshots (id, snapshot_id, terms_data, content_hash)

-- 同意記録
terms_agreements (id, reservation_id, snapshot_id, agreed_at, ip_address)

-- 施術記録
treatment_records (id, reservation_id, photos, memo, created_at)
